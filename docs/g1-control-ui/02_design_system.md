# Design system: AUROSY Skill Factory

Документ задаёт **согласованные принципы**, **дизайн-токены** и **набор компонентов** для веб-интерфейса платформы.

---

## 1. Принципы UX

1. **Визуал робота — в центре** там, где пользователь настраивает позу: 2D-схема и/или 3D, с подсветкой выбранной зоны тела.
2. **Человекочитаемые подписи по умолчанию:** короткие формулировки зон («левое плечо — вперёд/назад»), а не только `snake_case` и индексы.
3. **Экспертный режим** (таблица всех суставов, внутренние имена, индексы, переключение ° / rad) — вторичный слой.
4. **Единицы явно:** для авторинга keyframes — градусы с символом °; для reference-траекторий и отладки — радианы с подписью `rad`.
5. **Снижение когнитивной нагрузки:** не дублировать «кокпит» из десятков рядов чисел в основном потоке.

---

## 2. Визуальный характер

- **Контекст:** инженерный инструмент с элементами «studio»: тёмная тема, снижение бликов при длительной работе.
- **Акцент:** один основной акцентный цвет для первичных действий; **сигналы опасности/ошибки** — отдельный красный.
- **Реализация:** токены в [`web/frontend/src/styles.css`](../../web/frontend/src/styles.css).

## 3. Дизайн-токены

### 3.1 Цвет

- Семантика в коде опирается на **`--bg-default`**, **`--bg-elevated`**, **`--bg-surface`**, **`--text-primary`**, **`--text-muted`**, **`--color-primary`**, **`--color-destructive`**, **`--border-default`** (см. таблицу выше).
- **Статусы конвейера и валидации** — через компоненты DS и бейджи (успех/ошибка часто через **`--color-accent`** / **`--color-destructive`** и текст + иконку).

### 3.2 Типографика

- **Базовый размер UI:** 14–16px.
- **JSON / логи:** моноширинный шрифт (`ui-monospace`, `JetBrains Mono`, `Fira Code`).
- **Числа суставов:** выровнять по правому краю в таблицах.

### 3.3 Отступы и сетка

- Базовый шаг **4px** (0.25rem).
- **Desktop-first:** минимальная ширина для полного UI — 1280px; ниже — схлопывание sidebar.

### 3.4 Z-index

| Слой | Примеры |
|------|---------|
| base | Контент, схема робота |
| sticky | Закреплённый header таблицы |
| dropdown | Выпадающие меню |
| modal | Диалоги |
| toast | Глобальные уведомления |

### 3.5 Motion

- Переходы **150–250ms** для hover/focus панелей.
- Индикаторы «running» — циклическая анимация с `prefers-reduced-motion: reduce` → статичный индикатор.

## 4. Компоненты

Реализованы в [`web/frontend/src/components/ds/`](../../web/frontend/src/components/ds/):

| Компонент | Назначение |
|-----------|------------|
| **PageHeader** | Заголовок страницы, краткое описание, основное действие справа |
| **ValidateBanner** | Результат `POST /api/validate`: success / список ошибок |
| **EmptyState** | Нет данных: подсказка следующего шага |
| **PipelineStatusBadge** | idle / running / success / error |

Прочие компоненты (формы, таблицы, слайдеры) — в страницах и `src/components/`.

---

## 5. Доступность

- **Контраст:** текст на фоне не ниже WCAG AA для основного контента.
- **Фокус:** видимое кольцо `:focus-visible` на интерактивах.
- **Статусы:** не полагаться только на цвет — добавлять иконку или текст.
- **Слайдеры:** доступны с клавиатуры; где углы только для чтения — `aria-labelledby` и `aria-valuetext` (например слайдеры в Pose Studio / WASM).
- **`prefers-reduced-motion`:** ослабление анимаций тостов и индикатора «Выполняется».
